# 前端缓存清理指南

## 问题描述

浏览器报错: `ReferenceError: t is not defined`

这是因为浏览器缓存了旧的构建文件,而新的构建文件已经修复了这个问题。

## 解决方案

### 方案 1: 强制刷新浏览器 (推荐)

1. 打开浏览器访问 `http://127.0.0.1:5000`
2. 按下以下快捷键强制刷新:
   - **Windows/Linux**: `Ctrl + Shift + R` 或 `Ctrl + F5`
   - **Mac**: `Cmd + Shift + R`

### 方案 2: 清除浏览器缓存

#### Chrome/Edge:
1. 按 `F12` 打开开发者工具
2. 右键点击刷新按钮
3. 选择 "清空缓存并硬性重新加载"

#### Firefox:
1. 按 `Ctrl + Shift + Delete`
2. 选择 "缓存"
3. 点击 "立即清除"
4. 刷新页面

### 方案 3: 无痕/隐私模式

1. 打开浏览器的无痕/隐私模式:
   - **Chrome/Edge**: `Ctrl + Shift + N`
   - **Firefox**: `Ctrl + Shift + P`
2. 访问 `http://127.0.0.1:5000`

### 方案 4: 重启后端服务

如果上述方法都不行,尝试重启后端:

```powershell
# 停止当前运行的后端 (Ctrl + C)
# 然后重新启动
python -m api.main
```

## 验证修复

打开浏览器控制台 (F12),如果没有看到 `ReferenceError: t is not defined` 错误,说明问题已解决。

## 已修复的问题

1. ✅ 所有组件都已正确导入 `useTranslation` hook
2. ✅ i18n 配置已优化,禁用了 Suspense 模式
3. ✅ 添加了内联后备翻译资源
4. ✅ 前端已重新构建 (新的构建文件: `index-BbbQFNXm.js`)

## 如果问题仍然存在

如果清除缓存后问题仍然存在,请检查:

1. 浏览器控制台的完整错误信息
2. Network 标签中加载的 JS 文件名称是否为新的文件名
3. 后端是否正确提供了新的静态文件

可以在浏览器控制台运行以下命令检查:
```javascript
// 检查 i18n 是否正确初始化
console.log(window.i18n);

// 检查翻译函数是否可用
console.log(typeof window.i18n?.t);
```
