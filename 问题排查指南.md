# QueryWeaver 页面空白问题排查指南

## 当前问题

浏览器控制台报错：`ReferenceError: t is not defined`

这表明某个组件在 i18n 初始化完成之前就尝试使用翻译函数。

## 临时解决方案（快速恢复应用）

### 方案 1：禁用 i18n（最快）

1. **编辑 `app/src/main.tsx`**，注释掉 i18n 导入：

```typescript
import { createRoot } from "react-dom/client";
import "./index.css";
import App from "./App.tsx";

// 临时禁用 i18n
// import "./i18n";

// ... 其余代码保持不变
```

2. **重新构建前端**：
```bash
cd app
npm run build
```

3. **重启后端服务**

这样应用会显示翻译键（如 `common.loading`）而不是翻译后的文本，但至少能正常运行。

### 方案 2：使用开发模式（推荐用于调试）

不使用构建后的文件，直接运行开发服务器：

1. **启动前端开发服务器**：
```bash
cd app
npm run dev
```

这会在 http://localhost:8080 启动前端（带热重载）

2. **保持后端服务运行**：
```bash
# 在另一个终端
make run-dev
```

3. **访问** http://localhost:8080

开发模式下错误信息更清晰，更容易定位问题。

## 根本原因分析

### 可能的原因

1. **某个组件在顶层直接调用 `useTranslation()`**
   - 在组件外部或模块顶层使用了 `t` 函数
   - 需要确保所有 `useTranslation()` 都在组件内部使用

2. **i18n 异步初始化问题**
   - i18next 的 `init()` 是异步的
   - 某些组件在初始化完成前就渲染了

3. **循环依赖问题**
   - 某个模块导入了 i18n，同时 i18n 又导入了该模块

### 排查步骤

#### 步骤 1：检查是否有组件在顶层使用翻译

搜索项目中是否有类似的代码：

```typescript
// ❌ 错误：在组件外部使用
import { useTranslation } from 'react-i18next';
const { t } = useTranslation();

export const MyComponent = () => {
  return <div>{t('common.hello')}</div>;
};
```

应该改为：

```typescript
// ✅ 正确：在组件内部使用
import { useTranslation } from 'react-i18next';

export const MyComponent = () => {
  const { t } = useTranslation();
  return <div>{t('common.hello')}</div>;
};
```

#### 步骤 2：检查 i18n 相关的工具函数

查看 `app/src/utils/errorMapping.ts` 等文件，确保没有在模块顶层调用 `t` 函数。

#### 步骤 3：使用开发模式查看详细错误

开发模式会显示具体是哪个文件的哪一行出错。

## 长期解决方案

### 方案 A：等待 i18n 初始化完成

修改 `app/src/main.tsx`：

```typescript
import { createRoot } from "react-dom/client";
import "./index.css";
import App from "./App.tsx";
import i18n from "./i18n";

// 主题初始化代码...

const rootElement = document.getElementById("root");
if (!rootElement) {
  throw new Error("Failed to find the root element.");
}

// 等待 i18n 初始化完成
i18n.on('initialized', () => {
  createRoot(rootElement).render(<App />);
});

// 如果已经初始化，直接渲染
if (i18n.isInitialized) {
  createRoot(rootElement).render(<App />);
}
```

### 方案 B：使用 Suspense 包装

修改 `app/src/App.tsx`：

```typescript
import { Suspense } from 'react';

const App = () => (
  <Suspense fallback={<div>Loading...</div>}>
    <QueryClientProvider client={queryClient}>
      {/* ... 其他内容 */}
    </QueryClientProvider>
  </Suspense>
);
```

同时在 `app/src/i18n/index.ts` 中启用 Suspense：

```typescript
react: {
  useSuspense: true,  // 改回 true
  // ... 其他配置
}
```

### 方案 C：完全移除 i18n（如果不需要多语言）

如果项目暂时不需要多语言支持：

1. 移除所有 `useTranslation()` 调用
2. 直接使用中文文本
3. 删除 i18n 相关依赖
