# QueryWeaver API 后端启动指南

本文档提供了启动 QueryWeaver API 后端服务的完整步骤和配置说明。

## 快速启动

### 方法一：使用 Docker（推荐，最简单）

如果你只是想快速启动后端服务，使用 Docker 是最简单的方式：

```bash
# 1. 复制环境变量配置文件
copy .env.example .env

# 2. 编辑 .env 文件，配置必要的环境变量（至少需要配置 AI 模型的 API Key）

# 3. 使用 Docker 启动
docker run -p 5000:5000 --env-file .env falkordb/queryweaver
```

启动后访问：http://localhost:5000

### 方法二：本地开发环境启动（推荐用于开发）

如果你需要进行开发或调试，建议使用本地环境：

```bash
# 1. 安装依赖（后端 + 前端）
make install

# 2. 复制并配置环境变量
copy .env.example .env
# 编辑 .env 文件，配置必要的环境变量

# 3. 启动开发服务器（支持热重载）
make run-dev
```

或者手动启动：

```bash
# 使用 pipenv 启动
pipenv run uvicorn api.index:app --host 0.0.0.0 --port 5000 --reload
```

### 方法三：生产环境启动

```bash
# 1. 构建前端
make build-prod

# 2. 启动生产服务器
make run-prod
```

## 环境变量配置

在 `.env` 文件中，你需要配置以下内容：

### 1. 基础配置（必需）

```bash
# 应用环境：development（开发）、staging（预发布）、production（生产）
APP_ENV=development

# FastAPI 密钥，用于会话加密（必需）
FASTAPI_SECRET_KEY=your_super_secret_key_here
```

### 2. AI 模型配置（必需，三选一）

QueryWeaver 支持三种 LLM 提供商，你需要至少配置其中一种：

#### 选项 A：使用 Azure OpenAI（默认）

```bash
AZURE_API_KEY=your_azure_api_key
AZURE_API_BASE=https://your-resource.openai.azure.com/
AZURE_API_VERSION=2024-12-01-preview
```

#### 选项 B：使用 OpenAI

```bash
OPENAI_API_KEY=your_openai_api_key
```

#### 选项 C：使用 DeepSeek（推荐，成本低 90%）

```bash
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxx
DEEPSEEK_BASE_URL=https://api.deepseek.com
```

**DeepSeek 优势**：
- 成本比 OpenAI 低 90%
- 对中英文查询都有很好的支持
- OpenAI 兼容的 API 接口
- 支持国产化部署需求

**获取 DeepSeek API Key**：
1. 访问 [DeepSeek 平台](https://platform.deepseek.com/)
2. 注册账号（支持中国手机号注册）
3. 进入 API Keys 页面
4. 创建新的 API Key
5. 复制密钥（以 `sk-` 开头）

### 3. OAuth 配置（可选，用于用户登录）

如果需要用户登录功能，配置以下 OAuth 提供商：

#### Google OAuth

```bash
GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret
```

回调地址：`http://localhost:5000/login/google/authorized`

#### GitHub OAuth

```bash
GITHUB_CLIENT_ID=your_github_client_id
GITHUB_CLIENT_SECRET=your_github_client_secret
```

回调地址：`http://localhost:5000/login/github/authorized`

#### 微信登录

```bash
WECHAT_APP_ID=wx1234567890abcdef
WECHAT_APP_SECRET=your_wechat_app_secret
WECHAT_REDIRECT_URI=http://localhost:5000/login/wechat/authorized
```

**注意**：
- 需要在[微信开放平台](https://open.weixin.qq.com/)注册
- 生产环境需要 HTTPS
- 需要 ICP 备案（中国部署）

#### 企业微信登录

```bash
WECOM_CORP_ID=ww1234567890abcdef
WECOM_AGENT_ID=1000001
WECOM_CORP_SECRET=your_wecom_corp_secret
WECOM_REDIRECT_URI=http://localhost:5000/login/wecom/authorized
```

**注意**：
- 需要在[企业微信管理后台](https://work.weixin.qq.com/)配置
- 生产环境需要 HTTPS

### 4. 其他可选配置

```bash
# LLM 配置加密密钥（用于 Web UI 配置存储）
LLM_CONFIG_ENCRYPTION_KEY=your-32-byte-base64-encoded-key

# 对话记忆 TTL（秒），默认永久保存
MEMORY_TTL_SECONDS=604800  # 7 天

# 禁用 MCP 协议支持（默认启用）
DISABLE_MCP=false

# 自定义主机和端口
HOST=127.0.0.1
PORT=5000
```

## 前置依赖

### 必需的软件

1. **Python 3.12+**
   - 检查版本：`python --version`

2. **pipenv**
   - 安装：`pip install pipenv`

3. **FalkorDB 实例**
   - 使用 Docker 快速启动：
     ```bash
     make docker-falkordb
     ```
   - 或手动启动：
     ```bash
     docker run -d --name falkordb-test -p 6379:6379 falkordb/falkordb:latest
     ```

4. **Node.js 和 npm**（如果需要构建前端）
   - 检查版本：`node --version` 和 `npm --version`

启动成功后，你可以访问以下地址：

## 常见问题排查

### 问题 1：FalkorDB 连接失败

**症状**：启动时报错无法连接到 FalkorDB

**解决方案**：
```bash
# 启动 FalkorDB 容器
make docker-falkordb

# 检查容器是否运行
docker ps | findstr falkordb
```

### 问题 2：缺少环境变量

**症状**：启动时提示缺少必需的环境变量

**解决方案**：
1. 确保已复制 `.env.example` 到 `.env`
2. 检查 `.env` 文件中是否配置了必需的变量
3. 至少需要配置一个 AI 模型的 API Key

### 问题 3：端口被占用

**症状**：启动时提示端口 5000 已被占用

**解决方案**：
```bash
# 方法 1：使用不同的端口
PORT=8000 make run-dev

# 方法 2：查找并关闭占用端口的进程
netstat -ano | findstr :5000
taskkill /PID <进程ID> /F
```

### 问题 4：依赖安装失败

**症状**：`make install` 或 `pipenv sync` 失败

**解决方案**：
```bash
# 清理并重新安装
pipenv --rm
pipenv sync --dev

# 如果是 npm 依赖问题
cd app
npm ci
```

### 问题 5：OAuth "mismatching_state" 错误

**症状**：OAuth 登录时出现 CSRF 警告

**解决方案**：
- 开发环境：确保 `APP_ENV=development`
- 生产/预发布环境：确保 `APP_ENV=production` 或 `APP_ENV=staging`

## 开发模式 vs 生产模式

### 开发模式（Development）

```bash
make run-dev
```

特点：
- 支持热重载（代码修改后自动重启）
- 详细的调试日志
- 允许 HTTP 会话 Cookie
- 绑定到 127.0.0.1（仅本地访问）

### 生产模式（Production）

```bash
make run-prod
```

特点：
- 优化的性能
- 精简的日志
- 强制 HTTPS 会话 Cookie
- 绑定到 0.0.0.0（允许外部访问）
