## 问题原因

浏览器缓存了旧的 JavaScript 文件,导致仍然使用有问题的代码。

## 解决步骤

### 步骤 1: 清除浏览器缓存 (必须执行)

#### 方法 A: 硬刷新 (最简单)
1. 打开浏览器访问 `http://127.0.0.1:5000`
2. 按下快捷键:
   - **Windows**: `Ctrl + Shift + R` 或 `Ctrl + F5`
   - **Mac**: `Cmd + Shift + R`

#### 方法 B: 开发者工具清除缓存 (推荐)
1. 按 `F12` 打开开发者工具
2. 右键点击浏览器刷新按钮 (地址栏左侧)
3. 选择 **"清空缓存并硬性重新加载"** (Empty Cache and Hard Reload)

#### 方法 C: 手动清除缓存
**Chrome/Edge**:
1. 按 `Ctrl + Shift + Delete`
2. 选择时间范围: "全部时间"
3. 勾选 "缓存的图片和文件"
4. 点击 "清除数据"
5. 刷新页面

**Firefox**:
1. 按 `Ctrl + Shift + Delete`
2. 选择时间范围: "全部"
3. 勾选 "缓存"
4. 点击 "立即清除"
5. 刷新页面

### 步骤 2: 验证新文件已加载

1. 打开浏览器开发者工具 (F12)
2. 切换到 **Network** (网络) 标签
3. 刷新页面
4. 查找 JavaScript 文件,应该看到新的文件名:
   - ✅ `index-BbbQFNXm.js` (新文件)
   - ❌ `index-CaYRBG6m.js` (旧文件)

### 步骤 3: 检查控制台错误

1. 打开浏览器开发者工具 (F12)
2. 切换到 **Console** (控制台) 标签
3. 刷新页面
4. 检查是否还有 `ReferenceError: t is not defined` 错误

**如果没有错误**: ✅ 问题已解决!

**如果仍有错误**: 继续下一步

### 步骤 4: 使用无痕模式测试

1. 打开浏览器无痕/隐私模式:
   - **Chrome/Edge**: `Ctrl + Shift + N`
   - **Firefox**: `Ctrl + Shift + P`
2. 访问 `http://127.0.0.1:5000`
3. 检查是否正常工作

**如果无痕模式正常**: 说明是缓存问题,返回步骤 1 彻底清除缓存

**如果无痕模式也有问题**: 继续下一步

### 步骤 5: 重启后端服务

1. 停止当前运行的后端 (在终端按 `Ctrl + C`)
2. 重新启动后端:
   ```powershell
   python -m api.main
   ```
3. 等待后端启动完成
4. 刷新浏览器页面

### 步骤 6: 验证静态文件

检查后端是否正确提供静态文件:

1. 在浏览器访问: `http://127.0.0.1:5000/assets/index-BbbQFNXm.js`
2. 应该看到 JavaScript 代码,而不是 404 错误

3. 在浏览器访问: `http://127.0.0.1:5000/locales/zh-CN/common.json`
4. 应该看到 JSON 翻译数据

**如果看到 404 错误**: 说明静态文件路径有问题,需要检查构建输出

### 步骤 7: 检查构建输出

确认前端构建文件存在:

```powershell
# 检查构建目录
dir app\dist

# 应该看到以下文件:
# - index.html
# - assets\index-BbbQFNXm.js
# - assets\index-Du1jG3ei.js
# - assets\index-zMPE_Zs2.css
# - locales\zh-CN\common.json
```

**如果文件不存在**: 重新构建前端:
```powershell
cd app
npm run build
```

## 常见问题

### Q1: 清除缓存后仍然报错
**A**: 尝试以下方法:
1. 关闭所有浏览器窗口
2. 重新打开浏览器
3. 使用无痕模式测试

### Q2: 无痕模式也报错
**A**: 可能是构建问题:
1. 删除 `app/dist` 目录
2. 重新构建: `cd app && npm run build`
3. 重启后端服务

### Q3: 后端启动报错
**A**: 检查以下内容:
1. Python 环境是否正确
2. 依赖是否安装: `pip install -r requirements.txt`
3. 环境变量是否配置: 检查 `.env` 文件

### Q4: 翻译文件加载失败
**A**: 检查以下内容:
1. `app/dist/locales` 目录是否存在
2. 翻译文件是否存在: `app/dist/locales/zh-CN/common.json`
3. 后端日志是否有错误信息
