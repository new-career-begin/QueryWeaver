# QueryWeaver 产品开发指南

## 产品定位

QueryWeaver 是一个**开源的 Text2SQL 工具**,通过图驱动的模式理解技术,将自然语言问题转换为精确的 SQL 查询。

### 核心价值主张
- **降低数据访问门槛**: 让非技术人员也能查询数据库
- **提高查询效率**: 自动生成复杂的 SQL 查询
- **智能模式理解**: 基于图数据库的关系推理
- **对话式交互**: 支持多轮对话和上下文理解

### 目标用户
1. **业务分析师**: 需要快速获取数据洞察,但不熟悉 SQL
2. **产品经理**: 需要自助式数据分析能力
3. **数据工程师**: 需要快速原型和探索性分析
4. **开发者**: 需要集成 Text2SQL 能力到自己的应用

## 产品架构

### 技术栈
```
前端层
├── React 18 + TypeScript
├── Tailwind CSS + Radix UI
├── TanStack Query (状态管理)
└── Vite (构建工具)

后端层
├── FastAPI (Python 3.12+)
├── LiteLLM (多模型支持)
├── FalkorDB (图数据库)
└── Graphiti (对话记忆)

数据层
├── PostgreSQL (支持)
├── MySQL (支持)
└── FalkorDB (模式图存储)
```

### 核心功能模块

#### 1. 数据库连接管理
**功能描述**: 支持多种数据库的连接和管理

**用户故事**:
- 作为用户,我希望能够连接到我的 PostgreSQL/MySQL 数据库
- 作为用户,我希望能够管理多个数据库连接
- 作为用户,我希望能够查看数据库的表结构

**技术实现**:
- 支持 PostgreSQL 和 MySQL
- 安全的凭证存储
- 连接池管理
- 自动模式发现

**产品决策**:
- ✅ 优先支持 PostgreSQL 和 MySQL (市场占有率高)
- ✅ 使用 OAuth 进行用户认证
- ⏳ 未来支持: SQL Server, Oracle, MongoDB
- ❌ 不支持: 直接文件数据库 (SQLite, Access)

#### 2. 自然语言查询
**功能描述**: 将用户的自然语言问题转换为 SQL 查询

**用户故事**:
- 作为用户,我希望用自然语言提问,而不是写 SQL
- 作为用户,我希望系统能理解我的查询意图
- 作为用户,我希望能看到生成的 SQL 语句

**技术实现**:
- 基于 LLM 的查询理解
- 图数据库驱动的模式检索
- 向量嵌入相似度搜索
- 上下文感知的查询生成

**产品决策**:
- ✅ 支持中英文查询
- ✅ 显示生成的 SQL (透明度)
- ✅ 支持查询历史和上下文
- ⏳ 未来支持: 查询建议和自动补全

#### 3. 图驱动的模式理解
**功能描述**: 使用图数据库理解表之间的关系

## 设计原则

### 1. 渐进式披露
不要一次性展示所有功能,根据用户熟练度逐步展示高级功能。

```typescript
// 示例: 高级选项默认隐藏
const [showAdvanced, setShowAdvanced] = useState(false);

return (
  <div>
    <QueryInput />
    <button onClick={() => setShowAdvanced(!showAdvanced)}>
      高级选项
    </button>
    {showAdvanced && <AdvancedOptions />}
  </div>
);
```

### 2. 默认值优化
为常用场景提供合理的默认值,减少用户配置负担。

```python
# 示例: 合理的默认配置
DEFAULT_QUERY_LIMIT = 1000
DEFAULT_TIMEOUT = 30
DEFAULT_MEMORY_LENGTH = 5
```

### 3. 错误恢复
当出错时,提供明确的错误信息和恢复建议。

```typescript
// 示例: 友好的错误提示
if (error) {
  return (
    <Alert variant="destructive">
      <AlertTitle>查询失败</AlertTitle>
      <AlertDescription>
        {error.message}
        <br />
        建议: 请检查数据库连接或简化查询条件
      </AlertDescription>
    </Alert>
  );
}
```

### 4. 性能感知
让用户感知到系统在工作,避免"卡死"的感觉。

```typescript
// 示例: 流式响应和进度提示
const [status, setStatus] = useState("正在分析查询...");

useEffect(() => {
  const steps = [
    "正在分析查询...",
    "正在查找相关表...",
    "正在生成 SQL...",
    "正在执行查询..."
  ];
  
  // 模拟进度更新
  steps.forEach((step, i) => {
    setTimeout(() => setStatus(step), i * 1000);
  });
}, []);
```
