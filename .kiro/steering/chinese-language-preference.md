# 中文语言偏好指导

## 语言使用规范

### 主要交流语言
- **默认语言**: 简体中文
- **代码注释**: 使用中文注释，解释复杂逻辑和业务规则
- **文档编写**: 所有项目文档、需求文档、设计文档均使用中文
- **错误信息**: 用户面向的错误信息使用中文，便于理解
- **技术术语**: 技术术语可以保留英文，但需要提供中文解释

### 代码规范
- **变量命名**: 使用英文命名，但添加中文注释说明用途
- **函数命名**: 使用英文，函数文档使用中文
- **类和接口**: 使用英文命名，类文档使用中文描述

### 文档结构
- **README文件**: 使用中文编写
- **API文档**: 使用中文描述接口功能和参数
- **用户手册**: 完全使用中文，使用中文标点符号（，。！？：；""''）
- **开发文档**: 使用中文，代码示例保持英文，代码示例后提供中文说明

### 交流指导
- **代码审查**: 使用中文进行评论和建议
- **问题讨论**: 优先使用中文表达技术概念
- **需求澄清**: 使用中文确保准确理解业务需求

### 示例格式

#### 代码注释示例
```javascript
// 用户认证服务 - 处理登录和注册逻辑
class AuthenticationService {
    /**
     * 用户登录方法
     * @param {string} email - 用户邮箱地址
     * @param {string} password - 用户密码
     * @returns {Promise<Object>} 登录结果，包含用户信息和会话令牌
     */
    async login(email, password) {
        // 验证邮箱格式
        if (!this.validateEmail(email)) {
            throw new Error('邮箱格式不正确');
        }
        // 其他登录逻辑...
    }
}
```

#### 文档标题示例
- ✅ `用户认证系统设计文档`
- ✅ `API接口说明`
- ✅ `数据库设计规范`
- ❌ `User Authentication System Design`

### 特殊情况处理
- **技术术语**: 首次使用时提供中英文对照，如 "认证(Authentication)"
- **第三方库**: 保持原有英文名称，但添加中文说明
- **配置文件**: 使用英文键名，添加中文注释说明
- **通用惯例**: URL 和文件路径等通用惯例保持原文

### 质量标准
- 确保中文表达准确、专业
- 避免中英文混杂的表达方式
- 保持技术文档的严谨性和可读性
- 优先考虑中文用户的理解和使用体验
